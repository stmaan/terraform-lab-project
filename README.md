# 🚀 Terraform & Kubernetes Lab (2026) — Mission Accomplished!

Проект по созданию сквозного процесса управления инфраструктурой (IaC) и деплоя приложений. 

## 🏆 Наши достижения (Модуль 1 завершен)

Мы успешно прошли путь от «пустой консоли» до работающего приложения в Kubernetes:

### 1. Инфраструктура как код (IaC)
- **Terraform v1.15.1**: Установлен вручную и настроен для работы в условиях сетевых ограничений через зеркала (`.terraformrc`).
- **Модульная структура**: Проект разбит на `main.tf`, `variables.tf`, `providers.tf` и `outputs.tf` по стандартам индустрии.
- **State Management**: Инициализирован и поддерживается стейт для управления ресурсами.

### 2. Оркестрация (Kubernetes & Kind)
- **Kind Cluster**: Глубокое погружение в работу локального кластера (7 нод: 1 control-plane + 6 workers).
- **Immutable Deployment**: Победили `ImagePullBackOff`, перейдя на деплой через **SHA256 Digest** — это гарантирует 100% идентичность кода.
- **Deep Debugging**: Успешно проведена отладка портов (смена 5000 -> 80) через анализ логов контейнеров.

### 3. Приложение (Flask App)
- Развернуто в изолированном неймспейсе `terraform-lab`.
- Управляется через переменные окружения (`APP_ENV`).
- Доступно по порту **30085** внутри кластера.
